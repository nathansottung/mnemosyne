# Mnemosyne — User Handbook

A plain-language, task-based guide for a **careful novice** (a photographer or
family archivist, **not** a developer). If you're comfortable with code, the
project's top-level `README.md` and `docs/ARCHITECTURE.md` are the maintainer
reference instead.

Read these in order the first time:

1. [What is this? (start here)](00-what-is-this.md)
2. [Install it and take the first run](01-install-and-first-run.md)
3. [Set up safely](02-set-up-safely.md)
4. [Your first backup](03-your-first-backup.md)
5. [The 3-2-1 setup (make it bulletproof)](04-the-3-2-1-setup.md)
6. [Backing up to LTO tape](05-tape.md)
7. [Discs, mirror drives, and adopting old drives](06-discs-and-drives.md)
8. [Checking on your archive (the yearly ritual)](07-checking-on-your-archive.md)
9. [Getting your files back](08-getting-files-back.md)
10. [The Recovery Kit (your break-glass box)](09-the-recovery-kit.md)

Reference:

- [Glossary](glossary.md) — every technical word, in plain English.
- [Troubleshooting](troubleshooting.md) — symptom → cause → fix.

## Screenshots

Each guide ends with a **"Screenshots to capture"** checklist naming the images it
references. All handbook images live in [`../img/`](../img/) using the naming
pattern `<guide-number>-<step>.png` (for example `03-create-archive.png`). The
guides are complete and usable without the images; the checklists are a to-do list
for whoever captures them.

## For contributors

The writing rules for this handbook (plain language, defined terms, expected
results, the required closing sections, grounding every step in a real UI label)
are enforced and documented in [`../CONTRIBUTING.md`](../CONTRIBUTING.md).
